## 1. 디자인 방향

### 스타일
- **테마**: 다크 퍼플 (C 스타일)
- **배경**: `#0a0a18` ~ `#0d1b3e` 딥 다크 그라디언트
- **포인트 컬러**: `#a78bfa` (퍼플), `#fbbf24` (골드)
- **서비스명 타이포**: 퍼플→골드 그라디언트 텍스트, font-weight 800
- **카드**: `rgba(255,255,255,0.06)` 반투명 배경, `rgba(255,255,255,0.1)` 테두리, border-radius 10px
- **버튼**: `linear-gradient(135deg, #7c3aed, #6d28d9)` 퍼플 그라디언트

### 네비게이션
- **하단 고정 내비바** (position: fixed, bottom: 0)
- 탭 3개: 🎰 로또 | 🏦 연금복권 | 👤 마이
- 활성 탭: 상단 2px 보더 + 레이블 `#a78bfa`
- 기존 상단 메인탭 제거

## 2. 아키텍처

### 기술 스택 변경사항
- **Firebase Auth**: Google OAuth 로그인 (추후 카카오, 네이버 추가)
- **Firebase Firestore**: 사용자별 히스토리 동기화
- 기존 정적 구조(Netlify + 순수 HTML/CSS/JS) 유지
- Firebase SDK는 CDN으로 로드 (빌드 도구 없음)

### 데이터 구조 (Firestore)
```
users/{uid}/
  profile:
    displayName: string
    email: string
    photoURL: string
    createdAt: timestamp

users/{uid}/history/{docId}
  type: 'lotto' | 'pension'
  data: array        // 번호 배열
  createdAt: timestamp
```

## 3. 화면 구성

### 3-1. 메인 화면 (로또 / 연금복권)
- 기존 기능 전부 유지 (번호생성 옵션, 통계 분석, 히스토리)
- 상단 헤더: 서비스명 "대박이네" (그라디언트) + 상태칩
- 하단 내비로 탭 전환

### 3-2. 로그인 화면
- 경로: 마이 탭 진입 시 비로그인이면 로그인 화면으로 이동
- 구성:
  - 로고 아이콘 + 서비스명
  - 설명 문구: "로그인하면 번호 히스토리가 모든 기기에서 동기화됩니다"
  - **Google로 계속하기** 버튼 (활성)
  - 카카오로 로그인 버튼 (비활성, UI만 표시 — 추후 구현)
  - 네이버로 로그인 버튼 (비활성, UI만 표시 — 추후 구현)
  - "로그인 없이 둘러보기" 링크 (메인으로 이동)

### 3-3. 마이 페이지 — 로그인 상태
- 프로필 영역: 아바타(프로필 사진) + 이름 + 이메일
- 통계 요약 3칸 그리드: 총 생성 횟수 / 로또 생성 수 / 연금복권 생성 수
- 최근 생성 기록 목록 (Firestore에서 최신 10건)
- **로그아웃 버튼** (빨간 테두리 스타일)

### 3-4. 마이 페이지 — 비로그인 상태
- 계정 정보 영역 대신: 로그인 유도 배너 + **로그인하기** 버튼
- 히스토리 영역: 흐리게 처리 + "로그인 후 확인 가능합니다" 텍스트

## 4. 로또 6/45 세트 UX 개선

### 변경 내용
- **제거**: 생성 세트 수 stepper (1~10)
- **추가**: 1라운드 = 5세트 고정 노출, 세트별 자동/수동 토글

### 세트 상태
각 세트는 세 가지 상태를 가진다:
- **자동 (on)**: 자동 번호 생성 대상
- **수동 (on)**: 수동 번호 입력 대상
- **비활성 (둘 다 off)**: 생성에서 제외, 흐리게 표시

**기본값**: 전체 5세트 모두 자동 on

### 토글 동작 규칙
| 현재 상태 | 클릭 | 결과 |
|---|---|---|
| 자동 off | 자동 클릭 | 자동 on, 수동 자동 off |
| 자동 on | 자동 클릭 | 자동 off (비활성) |
| 수동 on | 자동 클릭 | 자동 on, 수동 off |
| 수동 off | 수동 클릭 | 수동 on, 자동 자동 off |
| 수동 on | 수동 클릭 | 수동 off (비활성) |
| 자동 on | 수동 클릭 | 수동 on, 자동 off |

### 자동 세트 동작
- "자동 번호 생성하기" 버튼 클릭 시
- 자동 on 세트에 한해 번호 생성
- 기존 핫/콜드넘버 옵션, 번호 고정/제외 설정 참조
- 생성 결과는 볼 형태로 세트 우측에 표시

### 수동 세트 동작
1. 수동 on 전환 시 6개 빈 입력칸 노출
2. 첫 번째 칸 자동 포커스 (`inputmode="numeric"` → 모바일 숫자 키패드)
3. 각 칸은 로또 번호 1개 (1~45, 최대 2자리)를 입력받는 `<input type="text">` 필드
4. 두 자리 입력 시 또는 키패드 '다음/완료' 버튼 입력 시 → 오른쪽 칸으로 자동 포커스 이동
5. 이미 입력된 칸 클릭 시 해당 칸 수정 가능
6. 6칸 모두 입력 완료 시: "번호 수정" 버튼으로 전환
7. 미완료 상태에서는 완료 처리 불가 (6칸 빈칸 없이, 1~45 범위 유효성 검사)
8. "번호 수정" 클릭 시 다시 입력 모드로 전환

### UI 레이아웃 (세트 1개)
```
[ 1세트 ]  [ 자동 ][ 수동 ]  [ ○ ][ ○ ][ ○ ][ ○ ][ ○ ][ ○ ]
                              ← 번호 볼 or 입력칸 6개 →
```
- 수동 입력 완료 후:
```
[ 1세트 ]  [ 자동 ][ 수동 ]  [ 4 ][ 18 ][ 25 ][ 31 ][ 37 ][ 43 ]
                                              [ ✏️ 번호 수정 ]
```
